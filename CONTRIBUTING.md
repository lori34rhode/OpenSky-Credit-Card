# Contributing to OpenSky Credit Card Tracker

Thank you for your interest in contributing to the OpenSky Credit Card Tracker project! Your support helps make this financial tracking utility more robust for users looking to build or rebuild their credit.

## Contribution Guidelines
To maintain code quality and ensure a smooth review process, please adhere to the following principles:
*   Write clean, readable, and well-documented JavaScript/React code.
*   Ensure UI components match the professional and clean aesthetic of modern financial tools.
*   Verify that any additions do not compromise user data privacy, as this app handles financial figures.

## How to Contribute
1.  **Fork** the repository on GitHub.
2.  **Clone** your fork locally: `git clone https://github.com/your-username/opensky-credit-card.git`
3.  Create a descriptive feature branch: `git checkout -b feature/amazing-new-feature`
4.  Commit your modifications with clear, concise messages: `git commit -m 'Add support for tracking interest rates'`
5.  Push your branch to your fork: `git push origin feature/amazing-new-feature`
6.  Open a **Pull Request** targeting the `main` branch of this repository.

## Reporting Issues
If you encounter a bug, security vulnerability, or functional glitch:
*   Check the existing repository Issues tab to see if it has already been reported.
*   If not, open a new issue using the **Bug Report** template.
*   Include steps to reproduce the error, your expected outcome, and screenshots if applicable.

## Suggesting Features
We love innovation! If you have ideas to optimize credit limit tracking or payment alerts:
*   Open a new issue and select the **Feature Request** template.
*   Clearly explain the problem your feature solves and how it benefits OpenSky Credit Card users.

## Code of Conduct
This project follows a strict professional standard. By participating, you agree to:
*   Foster a welcoming, inclusive, and harassment-free environment.
*   Be respectful of differing viewpoints and experiences.
*   Focus on constructive feedback for code improvements.

## Review Process
*   All pull requests must pass standard linting and the existing test suite.
*   Maintainers will review your code within 3–5 business days.
*   You may be asked to make structural or stylistic adjustments before final approval and merging.
